# Tests Folder

Эта папка предназначена для хранения результатов экспериментальных запусков и сравнительных тестов рекомендательных моделей.

## Назначение

Внутри `tests/` создаются отдельные подпапки для каждого сценария эксперимента.

Название подпапки должно быть информативным и отражать, какая логика или какая комбинация моделей тестировалась.

Примеры:

- `only_ALS`
- `ALS_ColdItem`
- `ALS_CatBoostRegressor`
- `ALS_MaxFusion`
- `ColdItem_UserFeaturesOnly`

## Что должно храниться внутри каждой папки эксперимента

Для каждого тестового сценария рекомендуется сохранять:

- CSV-файлы с результатами
- скриншоты метрик, графиков или итоговых таблиц
- при необходимости краткое текстовое описание запуска

## Рекомендуемая структура одной папки эксперимента

```text
tests/
└── only_ALS/
    ├── recommendations.csv
    ├── scored_pairs.csv
    ├── metrics.csv
    ├── result_screenshot.png
    └── notes.txt
```

## Какие файлы можно сохранять

Типовые CSV:

- `recommendations.csv` — итоговые top-k рекомендации
- `scored_pairs.csv` — все кандидатные пары с рассчитанными score
- `metrics.csv` — итоговые метрики по эксперименту
- `comparison.csv` — сравнение нескольких подходов

Типовые изображения:

- `result_screenshot.png`
- `metrics_plot.png`
- `comparison_table.png`

Типовые текстовые файлы:

- `notes.txt`
- `run_description.txt`

## Правила именования папок экспериментов

Рекомендуется использовать короткие и понятные названия:

- отражать модель или комбинацию моделей
- отражать тестовую гипотезу
- избегать слишком общих имён вроде `test1`, `final`, `new_try`

Хорошие примеры:

- `only_ALS`
- `ALS_vs_Regressor`
- `ALS_MaxFusion`
- `ColdItem_WithFeatures`

Плохие примеры:

- `test`
- `test2`
- `new`
- `final_final`

## Зачем нужна такая структура

Такая организация нужна для того, чтобы:

- быстро находить результаты конкретного эксперимента
- сравнивать разные подходы между собой
- не смешивать результаты разных запусков
- сделать проект понятным для преподавателя, коллеги или любого нового пользователя

## Практическая рекомендация

Один эксперимент — одна отдельная папка.

Если меняется хотя бы одно из условий:

- модель
- способ объединения score
- датасет
- набор признаков
- стратегия cold-item обработки

лучше создавать новую подпапку с отдельным названием.

## Итог

Папка `tests/` — это каталог всех экспериментальных результатов проекта.

Каждая вложенная папка должна отвечать на вопрос:

```text
Какой именно подход тестировался и какие результаты он дал?
```
